How to Backup a Website on S3

May 19th, 2012

Some Requirements First:

  • Free

  • Easy

  • Mirrors site

  • Interacts nicely with AWS S3 Storage

After a bit of extensive research I settled on a solution I found over at github called boto-rsync. The boto part communicates with Amazon and the rsync part…keeps a track of what has changed and what as not, by dumping files here and there at them it ask what is different and what is not.

Now, there were some challenges, I’m cheap and I’m on the lowest plan that my host had, and while it has the support for Python, the language of choice for boto, it does not play nice with other versions of Python that I have on the server. I don’t want mess with that configuration, even if I could, which I don’t think I can, but I didn’t try because it sounds like a bad idea, so the workout around

Install virtualenv - http://pypi.python.org/pypi/virtualenv

Now, I can whatever I would as long as start this up. http://pypi.python.org/pypi/boto_rsync

Here is the script that is run by a cron job.

#boots into the virtual env to get boto, boto then mirrors the directories via an rsync of sorts
cd /home/namename/users/.home/my_new_env/bin/
a. activate
boto-rsync -a UUUUUUUUUUUUUUUU -s IIIIIIIIIIIIIIIIIIIIIIIII  /home/adfadfadsf/users/.home/domains s3://c.namename.co

How to Save a Website

Apr 14th, 2012

What happened? The dude that had the full control over the site, just let it go. For reasons unknown to me, he did. The creator of the site, and other friends tried contacting him, and they got ignored. So, what to do. You have a site, not that small, not that large, and it’s dead. It’s embarrassing.

What we did before. Before all this I was working around all the problems with the lack SSH or even sFTP. I was mirroring the site once a week, using this snippet of code.

wget --mirror --ftp-user=user --ftp-password=pass -o /log.txt ftp://ftp.thelastpeople.com

That was fun, until I started shutdown my computer, but now that I have SSH enabled, I will post my experience how to back your site on Amazon and schedule via cron job on the server and hopefully, I will not delete the shell scripts for that as well.

I just needed the IP of the server, and I found it. Thanks for the cardinal rule of internet, which is something like nothing ever ever escapes the internet. So, I downloaded directory

scp ... /999.99.9999 /home/whatever/downloads

Then I acquired new server space and a new domain name

Moved everything back over, the entire file system, and all the SQL INSERT statements that I had mailed to me twice a day. Did a find & replace job on statements as the url changed.

All is fixed except file permissions which I changed with something like

find *.html | chmod 755

Then I fixed all the plugins or eliminated the plugins that didn’t handle the switch well.

Now, I just change the logs every now and then. Done. However, we have lost half of site traffic even though I updated the Google webmaster configuration.
